# 扮猪吃虎爽文一键生成器 Skill

## Overview

本 Skill 用于生成“扮猪吃虎”“逆袭打脸”“隐藏身份”类爽文，包括都市、医学神医、赘婿、玄幻废柴流、职场逆袭等题材。 

本 Skill 的目标是降低用户使用门槛：**只进行 1 轮选择式提问**，让用户从 4–5 个关键选项中快速选择，然后直接生成一章约 2000 字的爽文正文。

正文生成完成后，Skill 应询问用户是否需要继续生成视频版。如果用户需要，应将文字爽文转化为适合视频大模型使用的 **漫画风格 AI 短剧脚本、分镜和视频生成提示词**。如果当前平台具备视频生成能力，可优先调用平台的视频功能；如果不具备，则输出可复制到视频大模型中的完整分镜和提示词。

本 Skill 的完整流程：

```text
一轮选择 → 生成约 2000 字爽文正文 → 询问是否生成视频 → 若需要，输出漫画风 AI 短剧分镜和视频提示词
```

本 Skill 的重点不是简单完成“被轻视—反转”结构，而是强化爽文最核心的情绪落差：

```text
羞辱要深 → 压迫要足 → 反派要连续加码 → 主角越冷静越有压迫感 → 打脸要分层爆发 → 结尾要留下更大爽点
```

默认爽文节奏：

```text
主角低调入场
→ 被初步误判
→ 反派公开羞辱
→ 围观者站队嘲笑
→ 反派连续加码
→ 触碰主角底线
→ 主角冷静反问或出手
→ 第一层反转
→ 反派不服继续搬靠山
→ 第二层打脸
→ 全场震惊失声
→ 主角只露出冰山一角
→ 章末钩子
```

## Core Rule: One Round Only

除非用户主动要求深度设定，否则只问一轮。

这一轮必须同时收集：

1. 题材类型
2. 主角表面身份
3. 第一章打脸场景
4. 主角反转方式
5. 文风与爽点密度

用户选完后，**不要继续追问**，直接生成一章约 2000 字正文。

正文长度可在 1800–2600 字之间浮动。为了让羞辱和打脸更充分，允许略长于 2000 字。

## Video Adaptation Rules

### 1. 不要逐字照搬 2000 字正文

视频短剧应提炼核心戏剧冲突。  
优先保留：

- 主角被羞辱的名场面
- 反派最嚣张的一句台词
- 围观者嘲笑与沉默
- 主角冷静停顿
- 第一层事实打脸
- 第二层权威打脸
- 章末钩子

### 2. 视频节奏要更快

60–90 秒短剧不适合长段解释。  
每个镜头应有明确情绪功能：

```text
误判 → 嘲讽 → 加码 → 触底 → 静默 → 反转 → 震惊 → 二次打脸 → 钩子
```

### 3. 台词要短

每句台词最好不超过 20 个汉字。  
反派台词要嚣张，主角台词要冷静。

示例：

反派：

```text
“你也配坐这桌？”
“出了事，你负责得起吗？”
“跪下道歉，我还能放你一马。”
```

主角：

```text
“你确定？”
“现在停手，还来得及。”
“打这个电话，你会后悔。”
```

## Deep Humiliation Rules

前期羞辱不能只写一句“你也配”。必须通过场景、身份、围观者和现实压力，让读者替主角感到憋屈。

### 羞辱至少包含 3 层

生成第一章时，前期羞辱建议至少包含以下 3 层：

1. **身份羞辱**  
   反派基于主角表面身份否定主角，例如穷、低职级、废柴、赘婿、没背景、没资历。

2. **公开场合羞辱**  
   羞辱要发生在众人面前，让主角处于被围观、被误解、被嘲笑的位置。

3. **底线羞辱**  
   反派不能只攻击主角本人，还应触碰主角在意的人、事或原则，例如亲人、妻子、病人、师门、项目、尊严、承诺。

### 可选的额外羞辱层

根据题材可继续加入：

- **物质羞辱**：拿钱、衣着、座位、礼物、职位、身份牌羞辱主角。
- **资格羞辱**：质疑主角“不配进来”“不配说话”“不配坐这桌”“不配参与抢救”“不配上擂台”。
- **关系羞辱**：让主角被亲戚、同学、同事、岳家或宗门同门共同看低。
- **权力羞辱**：反派威胁开除、取消资格、影响考核、断绝资源、逐出家族或宗门。
- **替代性羞辱**：反派让主角倒酒、道歉、下跪、签字背锅、让出功劳、离开现场。

注意：可以写羞辱和压迫，但避免极端侮辱、仇恨表达或过度低俗内容。

## Escalation Rules

反派不能羞辱一次就被打脸。必须连续加码，让读者期待反转。

推荐加码节奏：

```text
轻视 → 嘲讽 → 公开否定 → 逼迫主角低头 → 触碰底线 → 搬出权力或靠山 → 主角反转
```

每章至少设置 2 次加码。如果篇幅允许，应使用 3 次以上加码。

---

## Dramatic Face-Slapping Rules

打脸不能只有一个结果。应尽量写成“分层打脸”或“连续反转”。

### 打脸至少包含 2 层

1. **事实打脸**  
   主角用事实证明反派判断错误，例如检查结果、合同、身份认证、测试结果、方案原件、现场结果。

2. **权威打脸**  
   更高层级人物、系统结果、权威机构、掌门、院长、总经理、客户、专家或反派靠山确认主角正确，进一步压垮反派。

推荐三段式打脸：

```text
第一段：主角轻描淡写指出破绽，反派不信。
第二段：现场证据出现，围观者开始动摇。
第三段：更高权威到场或结果确认，反派彻底失态。
```

---

## Emotional Payoff Rules

爽点不仅来自主角赢，还来自读者情绪释放。

### 打脸前必须让读者憋住一口气

在主角反转前，应至少安排一个“读者想让主角出手”的节点：

- 反派羞辱主角亲人。
- 反派耽误病人。
- 反派抢功后逼主角背锅。
- 反派让主角当众道歉。
- 反派让主角离开本该属于他的场合。
- 反派用权力威胁主角前途。
- 围观者集体误解主角。

### 主角反转时不能太急

主角出手前应有一个短暂的静默或停顿，让情绪蓄满。

### 主角不要急着解释自己

主角最有逼格的反转方式是：

- 不解释，直接让结果说话。
- 不争辩，只指出关键事实。
- 不炫耀，只淡淡反问一句。
- 不自证身份，让更高层人物替他证明。

## User Input Handling Rules

### 用户只给部分选择

如果用户只给 2–3 个选择，例如：

```text
医学神医，急诊抢救，番茄风
```

应自动补全其余设定，并直接生成正文。不要继续追问。

### 用户没有按格式回复

如果用户没有使用“1A，2B”格式，而是自然语言描述，也应自动解析。

例如：

```text
我想写医学神医，主角是规培医生，在急诊抢救时被主任看不起，最后一眼看穿病因。
```

应直接生成正文。

### 用户要求修改正文

如果用户对生成内容提出修改，例如：

- 更爽一点
- 打脸更狠
- 反派更嚣张
- 主角更冷静
- 加强章末钩子
- 改成短剧风
- 前面羞辱写得更足
- 后面打脸写得更戏剧性

应直接改写对应内容，不重新启动选项流程。

## Chapter Rhythm

单章建议采用以下比例：

- 前 45%：深度羞辱、误判、围观者站队、反派加码
- 中间 25%：触碰底线、主角沉默蓄势、第一层反转
- 后 20%：反派不服、搬靠山、第二层打脸
- 最后 10%：全场余震、主角冰山一角、章末钩子

每章必须包含：

- 一个明确场景
- 一个主角被低估的理由
- 至少三层羞辱或压迫
- 至少两次反派加码
- 一个主角冷静细节
- 一个触碰主角底线的节点
- 至少两层打脸
- 明显的围观者态度反转
- 一个章末悬念

## Common Scene Templates

### 同学聚会 / 宴会羞辱

深度羞辱建议：

- 安排主角坐边角位或被临时加椅。
- 反派拿衣着、收入、工作、家庭变故羞辱主角。
- 旁人跟着起哄，形成群体压迫。
- 反派让主角倒酒、道歉、让座或离席。
- 反派触碰主角亲人或旧事底线。

戏剧性打脸建议：

- 主角先不解释，只打一个电话。
- 酒店经理或集团高层到场，先否定反派资格。
- 众人才发现酒店、项目或宴会背后属于主角。
- 反派搬出靠山，靠山却对主角恭敬。
- 章末引出当年旧案线索。

### 急诊抢救 / 专家会诊

深度羞辱建议：

- 主角年轻、职位低，被主任或专家当众压制。
- 反派质疑主角“没资格说话”“出了事你负责吗”。
- 旁人沉默，护士或同事不敢支持主角。
- 反派威胁影响规培考核、职业前途。
- 患者状况恶化，主角底线被触碰。

戏剧性打脸建议：

- 主角先指出一个被忽略的关键指标。
- 主任不信，现场结果逐渐验证主角判断。
- 患者生命体征稳定，全场沉默。
- 外部专家或院长确认主角判断更高明。
- 专家认出主角曾参与的诊断系统或病例路径。

医学剧情应保持小说化表达，不提供现实可执行医疗指导。

### 家族逼迫 / 岳家羞辱

深度羞辱建议：

- 岳家当众称主角废物。
- 反派要求主角离婚、道歉、签协议、放弃财产或背锅。
- 妻子或亲人被牵连受辱。
- 家族长辈站队反派，让主角孤立无援。
- 反派拿权势、金钱或婚约压迫主角。

戏剧性打脸建议：

- 主角动用隐藏资源解决家族危机。
- 反派口中的靠山反而向主角低头。
- 岳家发现真正保住家族的人一直是主角。
- 原本羞辱主角的人反过来求他。
- 章末主角真实身份只露出一角。

### 宗门测试 / 擂台考核

深度羞辱建议：

- 主角被认为废柴，测试前被同门嘲笑。
- 反派天才公开要求主角滚下台。
- 长老偏袒反派，剥夺主角机会。
- 未婚妻或家族旧约被拿来羞辱。
- 反派逼主角承认自己不配修行。

戏剧性打脸建议：

- 测试石第一次异常但无人相信。
- 主角再次出手，显露隐藏血脉或远古传承。
- 长老震惊失态，反派天才道心动摇。
- 更高层宗门人物被惊动。
- 章末古老禁地或祖器产生反应。

### 公司会议 / 项目危机

深度羞辱建议：

- 主角被安排坐末位或不能发言。
- 上司抢走主角方案，还让主角背锅。
- 客户或高层当众质疑主角资历。
- 反派威胁开除、扣绩效、封杀行业机会。
- 项目失败会牵连主角团队或重要承诺。

戏剧性打脸建议：

- 主角拿出原始方案记录或关键证据。
- 客户发现真正解决问题的人是主角。
- 高层当场否定反派，要求主角接管项目。
- 反派试图狡辩，却被邮件、会议记录或系统日志二次打脸。
- 章末更大项目或幕后竞争出现。

## Style Options

### 番茄网文风，中爽

节奏快，对话多，情绪明确，每章有阶段性反转。羞辱和打脸都要清楚，不要过度文艺化。

### 短剧爽文风，极爽

节奏极快，冲突直接，反派更嚣张，打脸更密集。每 300–500 字要有一次明显情绪变化。

### 传统男频风，稳爽

主角沉稳，设定更厚，反转不油腻。羞辱可以更克制，但压迫感要足，打脸要有层级。

### 女频逆袭风，情绪爽

情绪张力强，羞辱与反击更重关系冲突。反转可通过身份、证据、资源、能力完成，重点写前后态度变化。

## Writing Rules

必须做到：

1. 开头尽快进入场景和冲突。
2. 主角出场要低调。
3. 前期羞辱要写足，不能一笔带过。
4. 反派要有表面优势。
5. 反派必须误判主角。
6. 反派至少连续加码两次。
7. 主角必须保持冷静。
8. 主角出手前要有情绪蓄势。
9. 打脸必须分层，至少有事实打脸和权威打脸。
10. 围观者反应必须明显。
11. 结尾必须留下悬念。
12. 主角真实身份不要一次性完全曝光。

避免：

1. 主角一开始就自报身份。
2. 主角长篇解释自己有多强。
3. 反派毫无理由地愚蠢。
4. 羞辱太浅，读者还没憋屈就反转。
5. 打脸太快，缺少证据、权威或场面反应。
6. 爽点重复，场景没有变化。
7. 围观者没有反应。
8. 主角过度嘴炮，降低逼格。
9. 背景设定太长，迟迟不进入冲突。
10. 章节结尾太平，没有追读欲望。

## Safety and Content Boundaries

生成内容时应避免：

1. 现实可执行的违法犯罪指导。
2. 详细暴力虐待描写。
3. 针对现实群体的仇恨或歧视表达。
4. 现实医学治疗建议伪装成专业指导。
5. 过度低俗、色情或侵犯性内容。
6. 真实个人隐私信息。

可以保留小说化冲突、虚构打斗、虚构权谋、虚构商业竞争，但不提供现实可操作的伤害性步骤。
